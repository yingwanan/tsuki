# tsuki

`tsuki` 是一款给 `Mizuki` 博客用户使用的 Android 写作应用。你可以直接在手机上写文章、改页面内容、调整站点配置，然后把内容发回自己的 GitHub 仓库，让博客自动更新。

## 你可以用它做什么

- 在手机上新建和继续编辑 Markdown 文章
- 一边写正文，一边预览排版效果
- 用快捷工具栏快速插入标题、列表、引用、代码块、链接等常用语法
- 补全文章标题、简介、日期、标签、分类、封面等文章信息
- 打开 GitHub 仓库中的已发布文章继续修改
- 直接编辑博客里的关于页、友链、项目、技能、时间线等配置内容
- 修改站点配置和功能页数据，不必再回到电脑上手动改文件
- 把文章、配置和页面内容一键同步到 GitHub 仓库

## 适合谁

如果你符合下面这些情况，这个应用就是为你准备的：

- 你正在使用 `Mizuki` 主题搭建博客
- 你的博客内容保存在 GitHub 仓库里
- 你希望在手机上完成写作、改配置和发布
- 你不想每次小改动都打开电脑处理

## 开始使用

### 1. 下载应用

你可以在 GitHub Releases 页面下载最新 APK：

- 仓库主页：<https://github.com/yingwanan/tsuki>
- 下载页面：<https://github.com/yingwanan/tsuki/releases>

### 2. 连接你的 GitHub 仓库

第一次打开后，进入应用里的设置页，依次填好：

- GitHub 用户名或组织名
- 仓库名称
- 分支名称
- 文章目录
- 页面目录
- 配置文件路径
- GitHub 访问令牌

填完以后，应用就知道该把文章和配置发到哪里。

### 3. 准备 GitHub 访问令牌

如果你用的是细粒度令牌，按下面这样选就可以：

- 只授权你的博客仓库
- 打开 `Contents` 的读写权限

如果页面里没有看到对应权限，一般说明你还没有选对仓库，或者创建的不是细粒度令牌。准备好后，把令牌复制到应用设置页即可。

### 4. 写文章

点进文章页后，你可以：

- 新建本地草稿
- 继续修改以前写到一半的内容
- 直接打开远端已发布文章继续编辑

正文支持编辑、分屏和预览三种模式。你可以先专心写，再随时切到预览确认排版。

### 5. 改页面和配置

除了写文章，你还可以直接在应用里：

- 修改关于页和其他 Markdown 页面
- 编辑友链、项目、技能、时间线、设备、番剧等数据
- 调整站点配置和功能页配置

这样你在手机上就能把博客内容和展示信息一起改完。

### 6. 发布到 GitHub

确认内容没问题后，点击上传即可。应用会把改动写回你的 GitHub 仓库，之后你的博客部署流程会按你原来的方式自动更新站点。

## 使用时你会看到的页面

- 文章页：管理本地草稿和远端文章
- 编辑页：写正文、预览排版、补全文章信息
- 配置页：编辑站点配置、功能页和页面内容
- 设置页：填写 GitHub 仓库信息和默认写作参数

## 安全说明

- GitHub 令牌只保存在你的设备本地
- 应用会使用 Android Keystore 做本地加密保护
- 你不需要把令牌交给任何第三方服务
- 请不要把有写入权限的令牌分享给别人
